The Shared Protocol

The seven moves

Both editions use the same operational spine. The difference is who sits in the foreman chair and which internal worker pool they can command.

1FrameWrite the finish line in one sentence.
2SplitCreate two to six non-overlapping lanes.
3AssignGive each agent a bounded lane and stop condition.
4KeepMain thread stays on the critical path.
5HarvestIntegrate, sharpen, reassign, or close.
6RepeatContinue until done or blocked by reality.
7SynthesizeClose with decision, verification, risks, findings.

Token Efficiency

Cursor is the catalyst

The economic win is routing. Use Claude and Codex for the expensive work they are best at: architecture, decomposition, judgment, inspection, and adjudication. Use Cursor Composer 2 as the coding-tuned implementation engine that turns bounded instructions into working files.

High leverage

2. Build With Composer 2

Cost engine

Cursor's coding-tuned lane does the high-volume implementation: applying precise plans, editing assigned files, wiring UI, and reporting changed files plus checks.

Practical pattern: Spend premium tokens on the plan. Spend Cursor execution on the code.

3. Inspect Before Accept

Quality gate

The foreman sends the result back through reviewers. This keeps Cursor fast without asking it to be the final judge of its own implementation.

The real trick

Swarm Dispatch turns model choice into a workflow decision. Cursor is not a backup worker; it is the implementation flywheel. Claude and Codex create the blueprint and quality gates. Cursor executes the bounded build. The foreman harvests, checks, and reassigns until the artifact is actually done.

The Model Currency Rule is the key Claude-side upgrade: call the current top model for each tier instead of hardcoding stale names.

Lean vs Deluxe

Claude defaults to Lean for routine work and escalates to Deluxe when missed bugs would cost trust, money, security, or a public release.

ModeUse WhenInspection Pattern
LeanRoutine patches, small changes, low-stakes builds.Builder → Opus inspector → foreman → fix planner → builder → final inspector.
DeluxePublic release, security/auth, production migration, payments, anything sleep-costly.Builder → parallel Claude Sonnet + Codex inspect → Opus adjudicates raw reports.

Deluxe Inspector Loop

Parallel first, adjudicate cold

The point is independence. If Inspector B sees Inspector A's report, you have anchoring bias. Deluxe keeps both reports raw, then uses Opus for judgment.

Proof Case

The bug that made the doctrine real

The origin project was an interactive map artifact. It proved why separated judgment matters: a visual regression appeared only at shorter desktop heights, exactly the kind of defect a builder can miss after staring at their own layout.

1440x900 overlap
1175x778 overlap
viewport gets shorter → timeline/control overlap risk rises

Why the inspector caught it

The builder had already made the map work. The inspector’s job was different: check the artifact under stress, find regressions, report before fixing, and let the foreman decide the repair lane.

This is the practical heart of Swarm Dispatch: the builder’s success is not the finish line. Independent inspection is the finish line gate.
