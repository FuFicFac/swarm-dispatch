Dispatch Loop

Dispatch means locking the goal, then routing work.

The foreman starts with a goal preflight, tracks which lane owns which work, harvests results, switches assignments, and keeps quality gates separate from implementation.

The Moves

0 GoalWrite finish line, done criteria, constraints, assumptions, and question boundaries.
1 FrameConfirm what must be true before the task is done.
2 SplitCreate two to six non-overlapping work lanes.
3 AssignGive each agent ownership, boundaries, and a stop condition.
4 KeepThe main thread stays on the critical path.
5 HarvestIntegrate, sharpen, reassign, or close each returned lane.
6 RepeatContinue until done or blocked by reality.
7 SynthesizeEnd with decision, verification, open risks, and material findings.

Why this matters

Without dispatch, parallel agents become parallel guesses. With dispatch, each worker has a job, and the foreman keeps the whole system moving.

Where Addy fits

Addy Osmani's agent-skills repo supplies the engineering method inside the lanes: spec, plan, build, test, review, simplify, and ship.
